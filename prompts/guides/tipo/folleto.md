# Type: Brochure / flyer (one-pager)

A short, persuasive, scannable document that promotes or summarizes something —
a book, a product, a service, an event, a course. Usually **PDF** (fast path with
your own `estilo_css`) or **DOCX** if the user wants to edit it. 1-3 pages.

## Recommended structure
1. **Header / hero** — strong title and a one-line hook (subtitle); the key
   identity (brand, author, "presented by"). A colored band reads well.
2. **What it is** — a tight paragraph: what it is and why it matters. No filler.
3. **Highlights** — 3-6 short bullets with the selling points / key facts.
4. **Body sections** — themed blocks (e.g. "Synopsis", "Who it's for", "What's
   included", "Schedule"). Keep each short; use subheadings.
5. **Call to action** — a clear next step in a highlighted box. Only put a **real**
   link/QR if you have one — never a placeholder.
6. **Footer** — source/credit line, edition, contact (only if real).

## Evidence-based rules
- **Visual hierarchy:** control what's seen 1st-2nd-3rd via size, color and
  contrast; group related items; separate with whitespace.
- **Hero headline:** the largest text on the page, near the top, **under 10 words**,
  bold and distinct.
- **Scannability:** short chunks, subheads, bullets — graspable in seconds.
- **One clear CTA:** action-oriented ("Order now", "Reserve your seat"), visually
  prominent (a contrast block), not intrusive.
- **Restraint:** minimal text; **max 2 fonts** and a **3-color palette**; one strong
  hero image (high resolution) that supports, not competes.
- Align everything to a grid to reduce clutter.

## Color palettes (pick one)
- **Bold / marketing:** `#FF4D4D` (primary), `#FFB020` (accent), `#1A1A2E` (text), `#FFFFFF` (bg).
- **Vibrant tech:** `#6D28D9` (primary), `#06B6D4` (accent), `#0F172A` (text), `#F8FAFC` (bg).
- **Elegant / premium:** `#1B2A41` (primary), `#C9A227` (gold accent), `#6B7280` (secondary), `#F5F3EF` (bg).

## Layout options
1. **Single-column hero:** full-width hero + giant headline on top → short value
   prop → 3 benefit points → CTA at the bottom. Best for emotional/brand impact.
2. **Multi-panel grid:** hero band on top, then 2-3 columns for
   features/services/pricing, with a CTA strip across the footer. Best for
   information-dense offerings.

## Best practices
- Pick one palette + one font pairing (`consultar_guia_diseno()`) and reuse them.
- Fill it with the **real content** from the source; a brochure is a finished
  piece, not a layout with blanks.

## Mistakes to avoid
- Walls of text — it must be scannable.
- Fake images, placeholder QR/links, or "space for image/logo here". Only real
  assets; if a cover image would help, **suggest it in the chat reply** and add it
  only if the user provides/accepts one.
- Reader tips or notes about the brochure itself inside the file — put those in chat.
- Blank fields ("add your contact here") unless the user asked for a fillable template.
