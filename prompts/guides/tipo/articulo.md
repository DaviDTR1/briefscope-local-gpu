# Type: Article / essay / blog post

Long-form prose that develops an idea or tells a story. Usually **PDF** or **DOCX**
(fast path, Markdown). The value is in the writing and the argument, so prose
dominates — not bullets.

## Recommended structure
1. **Title** — specific and inviting; optional subtitle/deck.
2. **Lead / introduction** — a strong hook in the first sentence; state what the
   piece is about.
3. **Body** — sections with descriptive subheadings (front-load the key words),
   each advancing one point. Mostly flowing paragraphs; a list only when it helps.
4. **Conclusion** — synthesize, don't just repeat; leave a takeaway.
5. **Optional** — byline/date, a pull quote, references/sources if relevant.

## Readability rules (evidence-based)
- **Measure 50-75 characters per line** (~66 ideal); set `max-width: 70ch`. Reading
  speed drops past ~80 CPL.
- **Body font ≥16px (18-21px preferred** for long-form), **line-height 1.5** (1.6-1.7
  if lines run long).
- **Short paragraphs:** 2-4 sentences, with clear spacing between them.
- **Subheadings every 2-3 paragraphs.**
- **Scannability:** ~79% of readers scan. Use bold keywords, the occasional list,
  and **pull quotes** as visual "speed bumps".

## Color palettes (pick one; reading-friendly)
- **Serif-on-cream (warm long-form):** bg `#FBF7EF`, ink `#2B2B26`, accent/links `#8A6D3B`, muted meta `#6E6A60`.
- **Modern sans (clean editorial):** bg `#FFFFFF`, ink `#1A1A1A` (~90% black, not pure), link accent `#2563EB`, captions/meta `#6B7280`.

## Layout options
1. **Single-column reading column:** centered, ~70ch wide, generous line-height —
   the default for essays and posts.
2. **Editorial with pull quotes:** same reading column plus styled `blockquote`
   pull quotes to break up long stretches.

## Best practices
- One palette + one font pairing; comfortable reading measure and line height.
- Coherent narrative thread between sections; smooth transitions.
- Use real facts and quotes from the source; attribute sources accurately.

## Mistakes to avoid
- Turning an essay into a bullet dump — it should read as prose.
- Inventing facts, quotes or sources.
- Placeholder images or "insert image here"; only real images, and if one would
  help, suggest it in chat.
- Reader-facing tips about the article inside the file — put those in the chat reply.
