# Type: Formal letter

Formal communication. Best in **DOCX** (editable) or **PDF** via
`generar_documento_markdown`. For a letterhead with a logo or brand design, use
`estilo_css` (PDF) or the code path.

## Structure
1. **Letterhead / sender** — name and details of the signer or organization (top).
2. **Place and date**.
3. **Recipient** — name, title, organization.
4. **Greeting** — "Dear …,". Address a named person whenever you know the name.
5. **Body** — first paragraph with the purpose; middle paragraphs with the detail;
   closing paragraph with the expected action or thanks.
6. **Closing** — "Sincerely," / "Best regards,".
7. **Signature** — name and title.

## Cover letter (carta de presentación)

A common case with its own conventions:

- **Length:** one page, **250-400 words** total.
- **Body in 3-4 paragraphs:** (1) opener — the role, where you found it and one
  compelling reason you fit; (2) evidence — top achievements tied directly to the
  job's requirements, quantified; (3) optional second evidence / culture-fit; (4)
  closing — reaffirm interest, thank them, suggest a next step.
- **Formatting:** 1-inch margins, single-spaced, one blank line between paragraphs,
  **no paragraph indentation**. Match the résumé's font and header so the two form
  a cohesive set.
- **Tone:** professional but authentic; emphasize relevance and problem-solving
  over generic pleasantries.

## Color palettes (pick one; restrained)
- **Classic:** Navy `#1F3A5F` letterhead/accent, Charcoal `#333333` text, White `#FFFFFF`.
- **Modern:** Slate `#2C5F7C` accent, Cool gray `#5A6473` meta, White `#FFFFFF`.

Keep color to the letterhead, the signer line and a thin rule — body text stays near-black.

## Layout options
1. **Block style (default):** everything left-aligned, no indentation, blank line
   between paragraphs. The standard for business letters.
2. **Branded letterhead:** logo + organization band at the top (code path or
   `estilo_css`), then the block-style body beneath.

## Best practices
- A single clear purpose per letter; courteous, professional tone matched to the recipient.
- Short paragraphs; get to the point in the first lines.
- Personalize with the specific details of the case (use `buscar_en_documentos` if
  the letter is based on the project).

## Mistakes to avoid
- Generic greeting or closing when you know the name.
- Mixing several unrelated subjects.
- A body longer than one page: if it exceeds one page, consider a report.
