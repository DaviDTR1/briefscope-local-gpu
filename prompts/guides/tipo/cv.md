# Type: CV / résumé

A concise, professional summary of a person's profile. Usually **PDF** (fast path
with your own `estilo_css`) or **DOCX** if it must be editable. One page is ideal,
two at most. When replicating an existing CV, keep its structure and apply only
the requested changes.

## ATS first (very important)

Most CVs are first read by an **Applicant Tracking System (ATS)** before a human.
Unless the user explicitly wants a creative/visual CV for a human-only review,
make it **ATS-safe**:

- **Single column.** Multi-column layouts get jumbled when the parser reads
  left-to-right across the page. Use two columns only for human-only/creative roles.
- **No tables, text boxes, images, icons, charts, headers or footers.** Never put
  the name, email or phone in a header/footer — many parsers ignore that zone.
- **Standard section headings** the ATS expects: *Experience / Work Experience,
  Education, Skills, Certifications* — not creative labels ("My Journey").
- **Web-safe single font:** Arial, Calibri, Garamond, Georgia or Times New Roman.
  Body 10-12pt, headings 14-16pt.
- **Standard dates**, reverse-chronological: `Jan 2023 – Mar 2025` or `01/2023 – 03/2025`.
- **Keywords:** mirror the exact terms from the job description in context (titles,
  summary, skills, first bullet of each role).
- **Format:** a clean text-based DOCX parses most reliably; PDF is fine for modern
  ATS as long as it is real text (not a scan/image). Offer DOCX when the user is
  applying online.

## Recommended structure
Contact → Professional summary (2-4 lines) → Skills (grouped) → Experience
(reverse-chronological: role, company, dates, 2-4 achievement bullets, prefer
measurable results) → Education → Optional (certifications, projects, publications,
awards).

## Color palettes (pick one; body text near-black on white)
- **Conservative (finance/law):** Navy `#1F3A5F`, Charcoal `#333333`, White `#FFFFFF`.
- **Modern professional:** Slate blue `#2C5F7C`, Cool gray `#5A6473`, White `#FFFFFF`.
- **Creative (human-only roles):** Teal `#2A9D8F` or Plum `#7B466A` accent + Charcoal `#2B2B2B`.

Use the accent only on the name, headings and dividers — never on body text.

## Layout options
1. **ATS-safe single column** (default): no tables/graphics, standard headings,
   ideally DOCX. For online applications.
2. **Two-column / creative**: sidebar for contact + skills, accent color, light
   icons — only when a human reviews directly (design/portfolio roles). Keep an
   ATS-safe single-column copy too.

## Best practices
- One palette + one font pairing; strong visual hierarchy; generous whitespace; easy to skim.
- Use only the real information given. If a section lacks data, **omit it** or note
  in chat that it could be added — do not invent entries.

## Mistakes to avoid
- Inventing jobs, dates, skills or contact details.
- Multi-column / tables / images on an ATS-bound CV.
- Placeholder photo boxes or "add your photo here" — add a real photo only if the
  user provides one; otherwise suggest it in chat.
- More than two pages, or dense paragraphs instead of crisp bullets.
